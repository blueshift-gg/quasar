# quasar-test

`quasar-test` makes an on-chain program test read like a normal Rust test:

```toml
[dev-dependencies]
quasar-test = "=0.1.0"
```

```rust,ignore
use {crate::cpi::DepositInstruction, quasar_test::prelude::*};

#[quasar_test]
fn deposits_into_the_vault(test: &mut Test) {
    let authority = test.add(Wallet::new());

    test.send(DepositInstruction {
        authority,
        amount: 1_000_000_000,
    })
    .succeeds()
    .cu_at_most(10_000);
}
```

`#[quasar_test]` is an ordinary `#[test]`: filters, `#[ignore]`,
`#[should_panic]`, and `Result<(), E>` work normally. `quasar test` builds the
program first and then runs the complete Cargo test graph. Direct `cargo test`
is also supported when a compiled artifact already exists in `target/deploy`.

Fixtures are values. `Wallet`, `Mint`, `TokenAccount`,
`AssociatedTokenAccount`, and `Program` cover common setup; applications can
implement `Fixture` once for protocol states and reuse them across tests.
Addresses generated by the world are deterministic per test. Legacy Token and
base Token-2022 accounts share the same fixture API.

Generated instructions derive canonical PDA, ATA, program, and sysvar accounts.
The normal `{Name}Instruction` asks only for caller-controlled values;
`{Name}InstructionRaw` exists for negative tests that deliberately override a
derived account.

`send` and `send_all` return `Outcome`, which owns stable error, log, return-data,
compute-unit, account, and account-change views. Generated client decoders plug
directly into `account_as`, `events`, and `return_value`. Missing writable
accounts enter the transaction as empty Solana accounts: QuasarSVM commits them
on success and leaves no placeholder after failure.

Every sibling program in `target/deploy` with a matching `-keypair.json` is
preloaded for CPI. `test.add(Program::new(id, elf))` is the explicit option for
an artifact outside that bundle.

The SVM remains an implementation detail. Tests depend on Solana's public
`Instruction`, `AccountMeta`, and `Pubkey` types plus `quasar-test`'s `Test`,
`Fixture`, `Account`, `Outcome`, and `ProgramError` contracts.
